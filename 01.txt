1)O que é o insumo, ou entrada, em um sistema de informação?
R:O insumo ou entrada é tudo oque entra no sistema para ser processado, o sistema só utiliza o necessário.
2)Onde podemos coletar dados para sistemas? Explique e cite exemplos.
R:Os dados são coletados com os usuários do sistema, com as pessoas que pediram a construção do sistema, com suas opiniões e sugestões.
3)Explique o que é um desenvolvedor backend, frontend, e full stack;
R: O desenvolvedor backend é aquele que fica com a parte da programação do sistema, com toda a parte burrocrática e inplementação das ideias.
   o desenvolvedor frontend é aquele que fica com toda a parte web do sistema, ele faz o sistema ficar bonito para o usuário, e tem como papel principal  deixar o sistema com uma boa usabilidade para o cliente.
   O desenvolvedor full stack é aquele que sabe fazer o backend e o frontend.
4)Qual a diferença entre frontend e designer?
R: O frontend é quem desenvolve a parte web do sistema.
   O designer cria as imagens para serem implementadas no sistema,a parte gráfica, deixando mais bonito e chamativo.
5)Quem é o responsável por manter um servidor de banco de dados e quais são suas obrigações?
R:O responsável é o DBA, que controla toda a parte do banco de dados, armazenando la os dados e garantido a segurança do armazenamento.
6)Que dados uma nota fiscal eletrônica possui? Cite todos e o objetivo de ter esses dados.
R: Ela registra eletrônicamente as operações tributárias dos empreendimentos,tem uma válidade jurídica que é assegurada pela assinatura digital da empresa emitente e pela autorização prévia de uso pela administração tributária do estado responsável pelo recolhimento do tributo,a troca da antiga nota fiscal pela nova nota fiscal eletrônica proporcionou inúmeras vantagens,como(a exclusão das notas de papel, a facilitando o dia a dia do trabalhador).Nela são armazenados todas as informações do cliente e do emitente, esses dados são armazenados online, no site da receita, ficando diponivel para pesquisas por ate 180 dias.
Ela reduz o impacto ambiental por nao ser mais de papel,reduz o tempo de entrega das mercadorias, diminui os custos com a impressão, incentiva o comércio eletrônico, aumenta a segurança e credibilidade, amplia o controle fiscal.
7)Quais os benefícios em versionar código fonte em ambientes como o GitHub?
R:Que no GitHub varias pessoa podem ver seus códigos, identificar possiveis erros e avisar o criador para corrigir o problema, assim todos se ajudam, porque ali podem ser compartilhados códigos fonte.

8)Como funciona o fluxo de trabalho e quais os benefícios em ter o trabalho dividido nas seguintes etapas: a fazer, fazendo, feito.
R: o fluxo de trabalho funciona de acordo com a necessidade do usuário, ele altera as etapas de suas tarefas, ele pode deixar tarefas a fazer, as tarefas que ele está fazendo no momento e as tarefas ja feitas por ele, isso deixa suas tarefas mais organizadas, e lembrando que o correto é fazer uma tarefa por vez, agilizando o seu dia a dia.
9)Como você imagina que seja a criação e desenvolvimento de um software? Explique detalhadamente cada atividade e porque ela acontece.
R: análize de requisitos: é a extração de dados do cliente.
   especificação:é a tarefa de descrever precisamente o software, de uma forma matemática rigorosa.
   arquitetura do software:é a representação abstrata daquele sistema, é a garantia que o software irá ao encontro dos requisitos do sistema.
   implementação:transforma um projeto para um código e deve ser a parte mais evidente do trabalho da engenharia de software.
   teste:se define como a parte de testes do sistema, mais específico na parte onde o software foi desenvolvido por mais de um engenheiro.
   documentação:a documentação é uma importante tarefa do projeto do software para propósitos  de futuras manutenções e aprimoramentos.
   suporte e treinamento:se define em treinar os usuários do sistema para que eles saibam como funciona o software e desenvolver um trabalho mais rápido, agilizando seu dia a dia.
   manutenção:se define em corrigir erros surgidos no sitema depois de instalado no cliente, buscando deixar sempre atualizado o sistema, sem falhas.
   
10)Explique para que serve e as principais funcionalidades do Trello.
R: No trello você adiciona quadros, com suas tarefas, você pode criar seu time, que irá lhe ajudar nas tarefas, ele possui três quadros( a fazer, fazendo, feito) la pode-se escolher as atividades que deseja fazer, lembrando que o correto é fazer uma por vez, para não embarralhar as tarefas.
11)Explique para que serve e as principais funcionalidades do Slack.
R: o slack serve para compartilhar seus códigos com seus amigos que estão programando junto, tambem serve como meio de se comunicar a respeito do software que está sendo desenvolvido ou corrigido, com uma facilidade incrivel,de forma totalmente segura, pode criar grupos para conversa como pode conversar individualmente.
12)Explique para que serve e as principais funcionalidades do Github.
R: o github serve para postar seus códigos, tanto em modo público com o privado, voce consegue fazer commits(atualizações no projeto), e repostalos novamente atualizados, várias empresas quando vão contratar alguém, eles olhar o github para analisar as informações postadas la,tambem no github é possivel baixar códigos de outras pessoas para usalo em seu projeto, dês de que a pessoa tenha postado em modo público.
